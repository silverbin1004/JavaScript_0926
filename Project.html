웹 브라우저가 html 문서를 해석할 때 문서 내용 전체를 트리 구조로 표현한 것. 자바스크립트를 이용해 각 요소에 접근할 수 있음. const title = document.getElementById('title'); 1. 'getElementById' => id를 이용하여 요소 노드 가져오는 방법
getElementById
const element1 = document.getElementByID('element1'); 2. 'getElementsByClassName' => class를 이용하여 요소 노드 가져오는 방법
getElementsByClassName
const element2 = document.getElementsByClassName('element2'); 3. 'querySelector' => css 선택자를 통해 요소 노드를 가져온는 방법
querySelector
const element3 = document.querySelector('.element3'); - BOM(Browser Object Model) : 자바스크립트가 브라우저와 소통하기 위해 만들어진 모델로 윈도우 객체 아래 속해있다. - window 객체 기능 : 1. 브라우저 제어 : 웹 페이지 경고창, 확인창 등을 표시할 수 있다 2. 브라우저 창 제어 : 현재 브라우저 창의 크기를 조절하거나 새 창을 열고 닫을 수 있다 3. url 조작 : 페이지의 url을 조회하거나 변경할 수 있다 - 사용자의 브라우저 히스토리에 접근할 수 있는 기능을 제공한다. CSS를 파싱하여 만든 스타일 구조체 브라우저는 html 파일 안에 있는 스타일과 html과 연결된 css 파일에 들어있는 스타일을 가지고 cssom 트리를 생성한다. 브라우저는 DOM생성 -> CSSOM 생성 -> 랜더 트리 생성 -> 레이아웃 단계 -> 페인팅 단계 -> 합성 단계 를 거쳐 랜더링 되는데 Reflow는 랜더링 과정 중 레이아웃 단계에서 레이아웃 배치, 크기 변경 등 시 발생하고 Repaint는 페인팅 단계에서 요소의 일부분에 수정이 있을 떄(ex. 색상 변경 등) 발생한다. 자바스크립트 실행 맥락 => 자바스크립트로 작성된 코드를 실행하는데 필요한 환경 또는 그 환경을 나타내는 객체를 말한다. 전역 실행 맥락은 자바스크립트가 실행될 때 만들어지고 함수 실행 맥락은 함수가 호출될 때 만들어진다. - 콜 스택 : 자바스크립트가 실행되며 생성되는 실행 컨텍스트를 저장하는 자료구조로, 가장 마지막에 호출된 함수가 가장 먼저 실행되는 FILO 스택 구조로 처리된다. 이벤트 루프는 콜 스택과 큐를 감시하며 비어있는 콜 스택에 작업을 넣는 일을 수행한다.
